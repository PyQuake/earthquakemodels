43 # How is the code organized
44 Most of the code were develop to be run in a python3 interpreter.
45 You may find that some files can be bash executed.
46 In most cases, you can call the most important methods in two ways.
47 The direct approach and the self-structured way.
48 
49 What we mean from direct approach: It is possible to simply call,
50 for example, the method to genarate a model by GA. That method needs
51 some arguments and you would have to provide them to be able to run the method.
52 
53 The self-structured way is an aprroach that allows you to call the
54 method in the right sequence making it easier to run the methods.
55 ```
56 
57 # Executing the main GA methods - self-structured way
58 ```bash
59 in applyGaModel.py you can generate some different kinds of GA models.
60 All similar methods follows the same parttern:
61  in case you want to create the GA model based on some ETAS ideas,
62  the you should run the following method:
63  execEtasGaModel(year, times, save=False)
64  you have to specify the year for the model and how many times you
65  would like to execute the GA. Also, you may chose to save the model
66  in a file. But first, you have to create the reference model for
67  comparison (in most cases it is the resulting data after filtering
68  the catalog by the year) by calling:
69  createRealModelforEtas(year, save=False)
70  with the same parameters, but times.
71 ```
72 
73 #Executing the main GA methods - The direct approach
74 ```bash
75 in the models.etasGaModelNP you can call the GA model method by itself.
76 You may do it by:
77  gaModel(NGEN,CXPB,MUTPB,modelOmega, year, n=500)
78 in which NGEN, CXPB, MUTPB and n are the GA parameters. modelOmega
79 is the reference model to be comapared (in most cases it is the
80 resulting data after filtering the catalog by the year). You should
81 choose the year parameter as well.
82 ```

89 one, execTests(year), executes all tests available for a group of
